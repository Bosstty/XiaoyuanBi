# 校园生活服务平台重设计方案

## 1. 产品定位

这不是一个“功能堆叠型工具箱”，而是一个围绕校园即时需求与互助协作建立的综合型移动应用。

核心认知必须在用户打开 App 的 3 秒内建立：

1. 这里可以发订单，找人代取代购。
2. 这里可以发任务，找同学接单协作。
3. 这里还有论坛内容，承接校园交流、经验分享和服务信息流。

因此，App 的主视觉与信息架构不应该平均分配权重，而应该形成清晰优先级：

1. 订单中心 / 代取代购
2. 任务中心 / 技能互助
3. 校园论坛 / 内容社区
4. 钱包、评价、个人中心作为支撑层

## 2. 用户角色融合

### 普通用户

- 发布代取代购订单
- 发布任务与查看接单情况
- 参与论坛内容互动
- 查看钱包、进度、评价、申诉

### 代取员用户

代取员不是独立 App，也不是完全独立模块，而是普通用户的增强身份。

代取员具备普通用户全部功能，并额外拥有：

- 抢单中心
- 在线 / 离线状态
- 服务区域管理
- 出库照片、送达照片上传
- 收益与接单评分管理

设计原则：

- 普通用户视角始终默认展示“生活服务平台”
- 一旦认证为代取员，在订单页、个人中心、首页动态区自然出现“接单能力”
- 不让代取员进入第二套系统，而是在同一套产品内无缝升级

## 3. 信息架构

### 底部导航

底部导航建议固定为 5 个主入口：

1. 首页
2. 订单
3. 任务
4. 论坛
5. 我的

说明：

- “订单”比“代取”更宽，后续可自然容纳抢单中心、配送进度、售后、申诉
- “任务”强调校园技能互助和协作收益
- “论坛”作为校园内容场，不与交易功能争抢第一层级

### 首页结构

首页不是静态欢迎页，而是“校园服务总控台”。

第一屏必须直接出现：

1. 订单中心入口
2. 任务中心入口
3. 实时动态摘要

建议模块顺序：

1. 顶部身份与状态条
2. 双主入口 Hero
    - 订单中心
    - 任务中心
3. 即时需求看板
    - 待接订单数
    - 今日新任务
    - 当前热门分类
4. 校园论坛精选
5. 我的进度 / 钱包 / 近期活动

### 订单页结构

订单页定位为“即时履约中心”，优先处理快递、外卖、药品、生活用品代购。

普通用户视角：

- 发布订单
- 查看订单状态
- 查看配送时间与照片凭证

代取员视角增加：

- 抢单大厅
- 订单筛选
- 在线状态
- 拍照上传节点

### 任务页结构

任务页定位为“校园技能与互助市场”。

强调：

- 分类清晰
- 报酬明确
- 完成时效明确
- 申请接单链路明确

### 论坛页结构

论坛不是传统 BBS 风格，而是“校园内容流”。

应承接三类内容：

1. 校园资讯与通知
2. 经验分享与技能文章
3. 服务相关讨论与互助信息

论坛在视觉上要更像内容 feed，而不是交易卡片列表。

## 4. 视觉设计方向

### 总体气质

关键词：

- 青春
- 可信
- 温暖
- 有效率
- 有校园真实感

不采用当前偏冷、偏硬、偏黑白的风格。

### 推荐风格

基于 `ui-ux-pro-max`，采用：

- Mobile app
- Soft editorial cards
- Campus-service dashboard
- Light mode first
- Warm blue + fresh green + sunrise orange accents

### 色彩系统

主色：

- Campus Blue: `#2F6BFF`
- Fresh Green: `#19B36B`
- Sunrise Orange: `#FF9B3D`

辅助色：

- Ink Navy: `#17304F`
- Lake Cyan: `#4BB8FF`
- Soft Gold: `#F7C75F`

背景色：

- Page Background: `#F4F7FB`
- Card Background: `#FFFFFF`
- Soft Block: `#EEF3FB`

文字色：

- Primary Text: `#172033`
- Secondary Text: `#5B667A`
- Tertiary Text: `#8C96A8`

### 视觉原则

1. 首页和订单/任务入口要有明显色块与卡片层次。
2. 论坛内容卡与订单卡、任务卡在样式上要可区分。
3. 避免大面积纯黑、纯白硬切割。
4. 通过渐变、圆角、柔和阴影体现校园服务平台的亲和力。

## 5. 页面设计策略

### 首页

目标：让人第一眼知道这里能“发订单、发任务、看动态”

视觉策略：

- 顶部采用渐变头图区
- 两个主入口使用大卡 Hero
- 下方用不同颜色模块区分订单、任务、论坛精选

### 订单页

目标：像“即时配送大厅”

视觉策略：

- 顶部强调当前可用订单和履约状态
- 卡片突出地点、时效、报酬
- 筛选条保持强可用性

### 任务页

目标：像“校园协作市场”

视觉策略：

- 分类色块 + 热门任务卡
- 任务卡突出预算、截止时间、申请热度
- 让技能互助的氛围比“跑腿感”更强

### 论坛页

目标：像“校园内容流”

视觉策略：

- 主打文章封面、板块标签、热度信息
- 板块入口可保留，但不应压过内容流
- 用更轻的卡片与更高的信息密度呈现文章

### 我的

目标：兼顾账户、钱包、进度、身份扩展

视觉策略：

- 用户身份卡
- 钱包 / 评分 / 订单 / 任务 统计
- 若为代取员，新增“接单中心”“在线状态”“收益”模块

## 6. 代取员扩展位设计

当前先预留，不在本轮做完整业务实现。

后续新增入口建议：

### 首页扩展

- 若已认证代取员，首页新增“抢单中心”快捷卡
- 顶部状态显示“在线 / 离线”

### 订单页扩展

- 增加“可抢订单”
- 增加“进行中”
- 增加“待上传凭证”

### 我的页扩展

- 增加“代取员中心”
- 增加“服务评分”
- 增加“今日收益 / 累计收益”

## 7. 交互原则

1. 所有主卡片可点击，且有明确触摸反馈。
2. 发布订单 / 发布任务按钮要始终易于触达。
3. 论坛内容卡点击进入详情，避免按钮过多。
4. 切换身份不需要切换产品，只改变入口与信息密度。

## 8. 本轮实现目标

本轮前端重设计优先完成：

1. 底部导航语义与视觉统一
2. 首页重构为校园服务总控台
3. 订单页重构为订单中心
4. 任务页重构为任务中心
5. 论坛页重构为校园内容流
6. 统一视觉变量与主卡片风格

暂不在本轮深入：

- 代取员抢单完整流程
- 图片上传业务链
- 复杂售后申诉流程
- 钱包与收益明细页完整实现

## 9. 实施原则

### 必须做到

- 首屏明确“订单 / 任务中心”
- 颜色符合校园服务平台气质
- 页面之间视觉关系统一
- 普通用户与代取员扩展逻辑可兼容

### 必须避免

- 黑白生硬、没有校园氛围
- 每页像不同产品
- 首页只起展示作用，不承接核心功能
- 代取员功能未来无法自然接入
